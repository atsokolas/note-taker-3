Noeis Documentation

How Noeis works: capture, think, and collaborate with agents

Noeis is a thinking system for people who read heavily and need reliable recall. It combines source capture, structured synthesis, semantic retrieval, and optional agentic workflows so ideas stay grounded in evidence and remain reusable over time.

Workflow in 4 steps

1) Capture

Bring material in through manual notes, direct paste, markdown files, Readwise CSV, or one-click clipping.

2) Shape in Think

Work in Notebook, Concepts, Questions, and Paths to convert raw highlights into structured thinking.

3) Delegate selectively

Use thought-partner prompts, concept suggestions, and handoffs while keeping explicit human approval.

4) Revisit and export

Use return queues, review loops, and full export controls so your knowledge stays portable and useful.

Main screen: Think

/think is the primary workspace. It is organized for fast continuation, not blank-page starts.

  • Home: Continue panel, working set (notes/concepts/questions), return queue, and recent material.
  • Notebook: Draft and edit notes, pull in highlights, and move text into working memory.
  • Concepts: Build concept notebooks, pin evidence, share read-only pages, and export markdown.
  • Questions: Track open loops and connect them to concept scope.
  • Handoffs: Route work between user, native agents, and personal BYO agents.
  • Paths: Follow connected concept paths across a topic.
  • Insights: AI-generated themes and concept connections when AI service is enabled.

Agentic capabilities (with guardrails)

Noeis supports agentic workflows, but keeps control with the user. Agents are assistive by default, with routing and approval controls exposed in product UI.

  • Thought Partner panel: Context-aware agent chat in Think (concept, notebook, question, or handoff context).
  • Concept suggestions: AI can propose highlights and drafts from your library; you choose what gets added.
  • Handoff queue: Create, claim, complete, reject, or cancel handoffs across actors.
  • Orchestration policy: Set defaults like balanced/native-first/BYO-first and task-type permissions.
  • Personal agents: Create named agents, rotate keys, and manage availability per workspace.
  • External bridge: Mint scoped, short-lived bridge tokens for A2A/MCP-compatible runtimes.

Settings page

/settings is both a workspace-control center and a trust page.

  • Workspace appearance: Typography scale, density, accent theme, and brand-energy toggle.
  • Onboarding controls: Restart onboarding and guided tour.
  • Agents & integrations: Quick setup for personal agents plus advanced integration controls.
  • Data integrations: Jump to import flows (manual notes, direct paste, markdown, Readwise CSV).
  • Export: Download JSON exports and PDF bundles to avoid lock-in.

Login and signup pages

Authentication is straightforward and keeps onboarding friction low while still enabling extension workflows.

  • Extension optional: Login/Register messaging clarifies the app works without browser extensions.
  • Session clarity: Expired sessions show an explicit relogin message.
  • Fast account flow: Register -> redirect to login -> enter Think without extra setup requirements.
  • Progressive setup: Users can start manually, then add extension capture or agents later.

Other areas that should stay aligned in marketing copy

  • Map + Review + Return Queue: Reinforcement and pattern-recognition loops after capture.
  • How-To page: In-app onboarding narrative for first-session behavior.
