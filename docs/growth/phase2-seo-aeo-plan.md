# Noeis Phase 2 SEO/AEO Plan

## Objective
- Move Noeis from a solid SEO foundation to a source-grounded, citation-friendly public surface that compounds qualified traffic into signups and activated users.

## Success metric hierarchy
- Primary:
  - non-branded qualified organic clicks
  - organic signup rate
  - organic activated-user rate
  - Bing AI citation visibility
- Secondary:
  - Search Console impressions and CTR on target clusters
  - assisted conversions from guide pages
  - return visits to public workflow pages

## Plan

### 1. Public surface upgrade
- Migrate the marketing surface toward true SSR or prerendered delivery.
- Keep every public growth page server-visible, snippet-eligible, and fast.
- Preserve one canonical URL per intent.

### 2. Citation-friendly page template
- Every public page should include:
  - direct answer near the top
  - visible byline and methodology
  - clear claim, evidence, and comparison structure
  - source-grounded examples
  - internal links by intent cluster
  - CTA tied to activation, not generic traffic

### 3. Intent-cluster publishing
- Core clusters:
  - recall vs capture
  - Readwise / import-heavy workflows
  - founders
  - researchers
  - AI for reading and synthesis
- Build pages that satisfy distinct query fan-out branches rather than generic category terms.

### 4. Public proof layer
- Publish workflow pages that demonstrate:
  - source to concept
  - concept to synthesis
  - reading archive import to first useful output
- Treat these as both ranking assets and product proof.

### 5. Measurement upgrade
- Measure:
  - visibility
  - citation
  - signup
  - activation
- Use:
  - Search Console
  - Bing Webmaster Tools AI Performance
  - internal funnel analytics

### 6. Control and hygiene
- Reduce duplicate and overlapping intent pages.
- Use canonicals, redirects, and internal links to consolidate authority.
- Use `data-nosnippet` selectively only where Noeis wants ranking without answer-surface reuse.

## Execution order
1. Publish source-grounded workflow proof pages.
2. Strengthen the page template and internal-link graph.
3. Upgrade public rendering architecture.
4. Connect Bing verification and real IndexNow.
5. Run the weekly operator loop on live Search Console and Bing exports.
